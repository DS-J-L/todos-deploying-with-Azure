# UI

## 1. UI 목표

PlanMate의 UI는 아래 원칙을 따른다.

- **날짜 맥락이 먼저 보일 것**
- **입력은 짧고 빠를 것**
- **월/주/일 전환이 혼란스럽지 않을 것**
- **반복 일정도 일반 일정처럼 자연스럽게 보일 것**

## 2. 디자인 방향

- 생산성 앱이지만 과하게 복잡한 업무용 도구처럼 보이지 않게 한다.
- 한 화면에서 “캘린더”와 “선택 날짜 상세 목록”을 함께 보여준다.
- 데스크톱에서는 2단 레이아웃, 모바일에서는 1단 레이아웃으로 전환한다.

## 3. 정보 구조

### 메인 화면
- 상단 툴바
- 캘린더 메인 영역
- 선택 날짜 상세 패널
- 일정 추가 / 수정 모달
- 토스트 메시지

## 4. 화면 정의

## 4.1 메인 캘린더 화면

### 상단 툴바 구성
- 앱 이름 / 로고
- 오늘로 이동 버튼
- 이전 / 다음 버튼
- 현재 기준 날짜 텍스트
- 뷰 전환 탭: `일 / 주 / 월`
- 새 일정 추가 버튼

### 메인 영역 구성
- 월별 뷰: 7열 달력
- 주별 뷰: 7일 행 또는 컬럼
- 일별 뷰: 단일 날짜 상세 목록

### 상세 패널 구성
- 선택 날짜 헤더
- 해당 날짜 일정 목록
- 완료/미완료 구분
- 수정/삭제 액션

## 4.2 일정 추가 / 수정 모달

### 필수 필드
- 제목
- 날짜

### 선택 필드
- 설명
- 반복 유형
- 반복 종료일
- 주간 반복 요일(매주 선택 시)

### 액션
- 취소
- 저장

### UX 원칙
- 제목과 날짜는 한 번에 눈에 들어와야 한다.
- 반복 설정은 숨겨진 복잡한 옵션이 아니라, 필요한 순간에 펼쳐지는 구조가 좋다.

## 4.3 모바일 화면

### 레이아웃
- 상단 툴바 최소화
- 캘린더가 우선 노출
- 상세 목록은 캘린더 아래에 위치
- 새 일정 추가 버튼은 하단 고정 또는 FAB 형태 허용

### 우선순위
1. 날짜 탐색
2. 일정 확인
3. 일정 추가

## 5. 뷰별 UI 규칙

## 5.1 월별 뷰
- 한 날짜 셀에는 일정 제목 최대 2~3개까지만 노출
- 그 이상은 `+N개` 형태로 축약
- 오늘 날짜는 배경 또는 테두리로 강조
- 선택 날짜는 별도 강조
- 완료 일정은 흐리게 또는 취소선 처리

## 5.2 주별 뷰
- 날짜별 카드 또는 컬럼에 일정 표시
- 주간 흐름이 보이도록 날짜 헤더를 강조
- 선택 날짜의 컬럼/행은 가볍게 하이라이트

## 5.3 일별 뷰
- 목록 중심 UI
- 일정 제목, 반복 여부, 완료 상태, 액션 버튼 노출
- 당일 일정이 많아도 스크롤 가능한 구조

## 6. 상태 디자인

## 6.1 로딩 상태
- 캘린더 본문 스켈레톤
- 상세 패널 스켈레톤 카드 2~4개
- 버튼은 비활성화 대신 최소한의 인터랙션 보장

## 6.2 빈 상태
- 문구 예시: `선택한 날짜에 일정이 없습니다. 새 일정을 추가해보세요.`
- CTA 버튼: `일정 추가`

## 6.3 에러 상태
- 문구 예시: `일정을 불러오지 못했습니다. 다시 시도해주세요.`
- 재시도 버튼 포함

## 7. 시각 스타일 가이드

### 톤
- 깔끔함
- 친근함
- 학습/개인 생산성에 맞는 차분한 UI

### 추천 스타일
- 라운드 카드
- 가벼운 그림자
- 과한 색상 사용 금지
- 상태 색상은 최소화
  - 기본
  - 선택
  - 완료
  - 에러

## 8. 반응형 기준

### Desktop
- 캘린더 + 상세 패널 2단 구성
- 툴바 전체 노출

### Tablet
- 2단 유지 가능
- 상세 패널 폭 축소

### Mobile
- 1단 구성
- 상세 패널은 캘린더 아래
- 버튼/입력 높이 충분히 확보

## 9. 접근성 기준

- 버튼과 입력 필드에 명확한 라벨 제공
- 키보드 포커스 이동 가능
- 체크박스/토글은 스크린리더에서 의미가 전달되도록 설계
- 색상만으로 완료 상태를 구분하지 않고 아이콘/텍스트 보조 사용
- 날짜 셀 클릭 영역 충분히 확보

## 11. MVP UI 결론

MVP에서 가장 중요한 UI는 아래 3가지다.

- 월/주/일 전환이 자연스러운 상단 툴바
- 캘린더 본문과 선택 날짜 상세 목록의 연결
- 반복 일정 입력이 과하게 어렵지 않은 일정 모달

이 세 가지가 잘 설계되면 앱의 핵심 경험은 충분히 살아난다.
